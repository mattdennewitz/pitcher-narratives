    ```python
    @dataclass
    class ExecutionMetrics:
        """Per-pitch-type execution metrics for the recent window."""
        pitch_type: str
        pitch_name: str
        csw_pct: float              # Called + Swinging Strike %
        zone_rate: float            # % of pitches in zones 1-9 (null zones excluded)
        chase_rate: float           # O-Swing%: swings on pitches in zones 11-14
        xwhiff_p: float | None      # From pitcher_type_appearance CSV
        xswing_p: float | None      # From pitcher_type_appearance CSV
        xrv100_p: float | None      # From pitcher_type_appearance CSV (window)
        xrv100_percentile: int | None  # Percentile vs all pitchers (from pitcher_type.csv)
        n_pitches: int
        small_sample: bool
        cold_start: bool

    @dataclass
    class AppearanceWorkload:
        """Workload data for a single appearance."""
        game_pk: int
        game_date: str              # ISO date string
        role: str                   # "SP" or "RP"
        ip: str                     # Baseball notation: "5.2", "1.0", "0.1"
        pitch_count: int
        rest_days: int | None       # None for first appearance

    @dataclass
    class WorkloadContext:
        """Workload and rest context for the pitcher."""
        appearances: list[AppearanceWorkload]
        max_consecutive_days: int    # Max consecutive calendar days pitched
        workload_concern: bool       # True when max_consecutive_days >= 3
    ```

    **New compute functions:**

    1. `compute_execution_metrics(data: PitcherData) -> list[ExecutionMetrics]`:
       - Get window_dates from _get_window_game_dates(data)
       - Check cold_start via _is_cold_start(data)
       - Filter statcast to window dates
       - For each pitch type in pitch_type_baseline (sorted by n_pitches desc):
         a. **CSW%**: Filter window statcast to this pitch type. Count pitches where description is in _CSW_DESCRIPTIONS. Divide by total pitches of that type. Multiply by 100.
         b. **Zone rate**: Filter window statcast to this pitch type AND zone is not null. Count where zone is in _ZONE_IN. Divide by total non-null-zone pitches. Multiply by 100.
         c. **Chase rate**: Filter window statcast to this pitch type AND zone in _ZONE_OUT. Count where description is in _SWING_DESCRIPTIONS. Divide by total outside-zone pitches. Multiply by 100. Return 0.0 if no outside-zone pitches.
         d. **xWhiff/xSwing**: Use _weighted_window_pplus-style pattern on pitcher_type_appearance CSV for window dates and this pitch type. Extract xWhiff_P and xSwing_P columns.
         e. **xRV100 percentile**: Compute pitcher's window xRV100_P (weighted from pitcher_type_appearance). Then load UNFILTERED pitcher_type.csv (via `data.agg_csvs["pitcher_type"]` is filtered to this pitcher -- need to reload full file). IMPORTANT: Load full pitcher_type.csv directly from data.AGGS_DIR / "2026-pitcher_type.csv" for the league distribution. Weight-average xRV100_P per (pitcher, pitch_type) across game_types. Count pitchers with HIGHER (worse) xRV100_P. Percentile = n_worse / total * 100. Use min_pitches=10 filter.
         f. **small_sample**: n_pitches < _MIN_PITCHES
       - Get pitch names from statcast pitch_type -> pitch_name mapping.
       - Return list sorted by n_pitches descending.

    2. `_compute_ip(statcast: pl.DataFrame, game_pk: int) -> str`:
       - Filter statcast to game_pk.
       - Get sorted unique innings.
       - For the final inning, count out-producing events (events in _OUT_EVENTS).
       - Add extra out for each grounded_into_double_play (GIDP = 2 outs).
       - Also count strikeout_double_play, sac_fly_double_play, double_play as 2-out events.
       - full_innings = len(innings) - 1 (all but last are complete)
       - total_thirds = full_innings * 3 + outs_in_final
       - Return f"{total_thirds // 3}.{total_thirds % 3}"

    3. `_compute_rest_days(appearance_dates: list) -> list[int | None]`:
       - Sort dates ascending.
       - First appearance returns None.
       - Subsequent: (date[i] - date[i-1]).days - 1

    4. `_max_consecutive_days(appearance_dates: list) -> int`:
       - Sort dates ascending.
       - Track consecutive run where (date[i] - date[i-1]).days == 1.
       - Return max run length.

    5. `compute_workload_context(data: PitcherData) -> WorkloadContext`:
       - Build list of AppearanceWorkload from data.appearances:
         - For each appearance row: compute IP via _compute_ip, get pitch_count from n_pitches column, compute rest_days.
       - Compute max_consecutive_days from appearance dates.
       - workload_concern = max_consecutive_days >= 3
