   - `window_release_x: float` (mean release_pos_x in window for this pitch type)
   - `season_release_x: float` (mean release_pos_x full season for this pitch type)

   - `pitch_types: list[ReleasePointPitchType]`
   - `cold_start: bool`

3. Create `def compute_release_point_metrics(data: PitcherData) -> ReleasePointMetrics`:
   - Use `_is_cold_start(data)` and `_get_window_game_dates(data)` and `_build_name_map(data.statcast)`
   - Filter `data.statcast` to window dates, drop rows where release_pos_x/z/extension are null
   - Group by pitch_type, compute mean release_pos_x, release_pos_z, release_extension for window
   - Do the same for full season statcast
   - For each pitch type present in both window and season, compute delta strings using `_movement_delta_string` (release point changes are measured in the same unit as movement -- inches/feet)
   - Use threshold of 0.1 ft for release_x and release_z deltas (more sensitive than movement -- 0.1 ft = ~1.2 inches, meaningful for release point), and 0.2 ft for extension
   - NOTE: Create two small private helpers `_release_delta_string(delta, threshold=0.1)` and `_extension_delta_string(delta, threshold=0.2)` that format as feet with 2 decimal places -- e.g. "Up 0.15 ft" / "Steady (+0.05 ft)". Follow the same pattern as `_movement_delta_string` but with ft units.
   - Order results by season pitch count descending (to match arsenal ordering)
   - Handle cold start: all delta strings get `_COLD_START_STRING`

4. Add `ReleasePointPitchType`, `ReleasePointMetrics`, `compute_release_point_metrics` to `__all__`.
